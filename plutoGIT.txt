▎Шаг 1: Создание нового репозитория на GitHub
1. Перейдите на GitHub (https://github.com/).
2. Нажмите на кнопку "New" или "Создать репозиторий".
3. Заполните поля:
• Repository name: имя вашего репозитория.
• Description: (опционально) описание вашего репозитория.
• Выберите, будет ли репозиторий публичным или приватным.
4. Нажмите "Create repository".
▎Шаг 2: Создание локального репозитория
Откройте терминал и выполните следующие команды:
# Переход в директорию, где вы хотите создать проект
cd путь/к/вашей/папке
# Создание новой директории для вашего проекта
mkdir имя_вашего_проекта
cd имя_вашего_проекта
# Инициализация нового репозитория Git
git init
▎Шаг 3: Добавление файлов
Создайте файл (например, README.md) и добавьте в него немного текста:
# Создание файла README.md
echo "# Имя вашего проекта" > README.md
▎Шаг 4: Добавление файлов в индекс Git
Теперь добавьте файл в индекс:
# Добавление всех файлов в индекс
git add .
▎Шаг 5: Создание коммита
Теперь создайте коммит с описанием изменений:
git commit -m "Первый коммит: добавлен README.md"
▎Шаг 6: Связывание локального репозитория с удалённым
Скопируйте URL вашего репозитория с GitHub (он будет выглядеть примерно так: https://github.com/ваш_логин/имя_вашего_репозитория.git) и выполните следующую команду:
# Установка удалённого репозитория
git remote add origin https://github.com/ваш_логин/имя_вашего_репозитория.git
▎Шаг 7: Пуш изменений на GitHub
Теперь вы готовы отправить изменения на GitHub:
# Отправка изменений в ветку main (или master, в зависимости от настроек)
git push -u origin main
▎Полный код
Вот полный набор команд, который вы можете скопировать и вставить в терминал:
# Переход в директорию, где вы хотите создать проект
cd путь/к/вашей/папке
# Создание новой директории для вашего проекта
mkdir имя_вашего_проекта
cd имя_вашего_проекта
# Инициализация нового репозитория Git
git init
# Создание файла README.md
echo "# Имя вашего проекта" > README.md
# Добавление всех файлов в индекс
git add .
# Создание коммита с описанием изменений
git commit -m "Первый коммит: добавлен README.md"
# Установка удалённого репозитория
git remote add origin https://github.com/ваш_логин/имя_вашего_репозитория.git
# Отправка изменений в ветку main
git push -u origin main
▎Примечания
• Если ваша основная ветка называется master, замените main на master в команде git push.
• Если вы используете двухфакторную аутентификацию на GitHub, 
вам может понадобиться использовать токен доступа вместо пароля при выполнении команды git push.
